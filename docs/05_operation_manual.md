# 05. 運用マニュアル

## 5.1 起動
```bash
make api            # http://127.0.0.1:8077/ui/  （API Docs: /docs）
```
本番は Docker 常駐（`docker-compose up -d`）。

## 5.2 管理UIの使い方（React + TS、7タブ）
- **ダッシュボード**：状態・バックボーン・索引サイズ・判定内訳。デモ操作（生成＆学習/索引再構築）。
- **照合**：入力は3通り — ①サンプル撮影取得、②ファイル選択、③**カメラで撮影**（PC/モバイルの
  カメラから直接、`getUserMedia`）。（任意で期待品番を変更）→「照合する」。**識別モード**
  （期待品番なしで品番推定）にも対応。結果は OK(緑)/NG(赤)/REVIEW(黄)/RETAKE(灰) で色分けし、
  Top候補・信頼度・画質を表示。
  - カメラはセキュアコンテキスト必須：PCは localhost で可、モバイルは HTTPS 起動（`make api-https`）が必要。
- **品番マスタ**：品番の**新規追加・編集**（名称/カテゴリ/類似グループ/**品番別しきい値**/状態）。
- **履歴・レビュー**：照合ログを判定で絞り込み。各行「正/誤」でフィードバック登録（再学習データ化）。
- **精度レポート**：稼働中モデルの**自己診断**（誤受理率・OK精度・取り違え検出率・レイテンシ）を実行。
- **画質調整**：画質しきい値の**自動キャリブレーション**（導出・適用・既定へリセット）。
- **設定**：判定しきい値（accept/review/margin/similar/top_k）の**実行時調整**、モデル・索引情報。

## 5.3 新規品番の追加（再学習なし）
1. 品番マスタへ登録（`POST /api/v1/parts`）。類似グループがあれば `group_id` を設定。
2. 撮影手順書に従い複数枚撮影し `var/data/raw/<part_no>/` に配置。
3. `POST /api/v1/build-index`（または UI「索引再構築」）で embedding を追加。
   - 距離学習の投影ヘッドは流用可。大幅な分布変化が無い限り再学習不要。

## 5.4 しきい値チューニング
- 全体既定は `.env`（accept/review/margin/similar_margin）。
- **類似品が多い高リスク品番**は品番別しきい値で厳しめ（accept↑, margin↑）に。
- 変更は `POST /api/v1/parts` で即時反映（エンジンが品番別しきい値を再読込）。

## 5.4.1 画質しきい値のキャリブレーション
導入時・撮影環境変更時は、現場の良品画像でしきい値を再導出する。
```bash
python scripts/40_calibrate_quality.py --good-dir <良品> --bad-dir <不良> --save
```
保存すると `MatchEngine` が起動時に `var/models/quality_thresholds.json` を自動適用する。

## 5.5 日常運用フロー（現場）
```
撮影 → [RETAKE]なら再撮影
     → [OK]   出荷可（コンベア前進）
     → [REVIEW] 作業者が Top候補を目視確認 → 正/誤をフィードバック
     → [NG]   出荷停止・排出・原因確認（取り違えロット隔離）
     → [UNKNOWN] 未登録疑い → 品番登録 or 確認
```

## 5.6 精度監視と継続改善
- ダッシュボード/`GET /api/v1/stats` で OK率・REVIEW率・NG率・平均処理時間を監視。
- 週次で `feedback_logs` を集計し、誤判定・要確認が多い品番を特定。
- 蓄積データで投影ヘッドを再学習（`scripts/20_build_index.py --epochs N`）。
- 取り違え多発の類似グループは撮影条件強化（複数角度・専用照明・治具）。

## 5.7 バックアップ・復旧
- DB（`var/partmatch.db` / 本番PostgreSQL）と索引（`var/models/index.*`）、投影ヘッド
  （`projection.npz`）、登録画像（`var/data/raw`）を定期バックアップ。
- 復旧は DB復元 + `build-index` で索引再生成（画像が残っていれば完全復元可）。

## 5.8 障害対応
| 事象 | 対応 |
|------|------|
| API 応答なし | C#側フェイルセーフ（自動OK禁止→要確認）。プロセス/GPU/メモリ確認、再起動 |
| 精度低下 | 撮影条件変化を疑い画質メトリクスを確認。照明/治具点検、しきい値見直し |
| 索引未構築(409) | `build-index` を実行 |
| 特定品番だけ誤判定 | 追加撮影・品番別しきい値・類似グループ設定を見直し |
