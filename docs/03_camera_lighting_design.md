# 03. 撮影環境設計（カメラ・照明）

> AI精度の大部分は撮影条件で決まる。PoC段階では開発者が撮影環境設計とサンプル撮影を主導し、
> 本番では現場担当者が同じ品質で撮影・登録できる「仕組み」を構築する。

## 3.1 金属プレス部品の難しさ
- 鏡面反射（テカリ）で見え方が変わる
- 角度・高さのわずかな差で陰影が変化
- 似た外形が多く、穴位置・ノッチなど微小特徴で識別する必要

## 3.2 カメラ選定の指針
| 項目 | 推奨 |
|------|------|
| 種別 | 産業用エリアカメラ（USB3 Vision / GigE Vision） |
| メーカー候補 | Basler / OMRON / KEYENCE / Cognex 等 |
| 解像度 | 識別したい最小特徴（穴・刻印）が十数px以上で写る画素数 |
| レンズ | 固定焦点・低歪み。必要に応じマクロ |
| 補助 | 偏光フィルタ（反射抑制）、無反射背景、位置決め治具 |

## 3.3 照明設計
| 方式 | 特徴 | 適性 |
|------|------|------|
| ドーム照明 | 拡散均一光・反射ムラ最小 | 金属面の安定撮影に最適 |
| 同軸落射 | 正面均一・刻印を出す | 平面部の刻印/印字 |
| リング | 安価・簡便 | 反射が出やすく金属は注意 |
| バー/ローアングル | エッジ・段差強調 | 外形・段差の識別 |
| 偏光（クロスニコル）| 鏡面反射を除去 | テカリの強い部品 |

原則：**反射を抑え、形状・穴・刻印のコントラストを最大化**する。1方式で難しければ複数照明・
複数角度を併用（docs/07 のマルチビュー）。

## 3.4 撮影条件の標準化（再現性）
- 部品の向きを治具で固定（姿勢ばらつきを最小化）
- カメラ距離・角度・露出・照明を固定し**撮影手順書**化
- 背景は無地・無反射で部品とコントラストを確保
- 撮影条件は `capture_conditions` テーブルに記録し、画像と紐付ける

## 3.5 画質ゲート（本実装 `quality/checks.py`）
撮影直後に以下を自動検査し、不合格は照合せず **RETAKE**：
- ピンボケ（ラプラシアン分散）
- 露出（平均輝度・白飛び・黒つぶれ）
- コントラスト不足（無地・部品不在）
- 鏡面反射（高輝度比率）
- 前景被覆率（不在・寄りすぎ）
- 位置ズレ（重心の中心からの偏差）

> しきい値は **現場の実画像で必ず再キャリブレーション**すること（`quality/checks.py: DEFAULTS`）。

### 画質しきい値の自動キャリブレーション
良品画像（と任意で不良画像）の分布からしきい値を自動導出するツールを実装済み。

```bash
# 現場画像で導出・保存（保存すると MatchEngine が起動時に自動適用）
python scripts/40_calibrate_quality.py --good-dir <良品> --bad-dir <不良> --save
# 実画像が無い場合のデモ（合成の良品/劣化画像）
python scripts/40_calibrate_quality.py --demo
```

合成デモでの実証結果: **良品の誤棄却率 0.83% / 不良（ピンボケ・露出・反射・位置ズレ）の棄却率 100%**。
保存先は `var/models/quality_thresholds.json`（`PARTMATCH_QUALITY_THRESHOLDS_PATH`）。

## 3.6 撮影・登録の役割分担（推奨）
| 作業 | 担当 |
|------|------|
| 撮影環境設計・カメラ/照明選定・治具仕様 | 開発者（主導） |
| PoCサンプル撮影・画質基準作成・手順書 | 開発者（主導） |
| 2万品番の量産撮影・新規品番の登録 | 現場担当/クライアント（手順書に従う） |
| 画質チェック・データ設計・学習・精度改善 | 開発者 |

開発者が全2万品番を撮影する必要はないが、**AI側から見た必要条件（撮影品質基準）**は開発者が定義する。
