# 06. 開発計画（PoC→本番）

> 見積もり（金額）は本ドキュメントの対象外。技術的なフェーズ計画とマイルストーンを示す。

## 6.1 段階的アプローチ
最初から2万品番を狙わず、撮影条件を固定できるかを早期に確認し、数百品番のPoCで実現性を実証、
MVPで現場フローへ接続、本番で2万品番・1秒以内・高信頼運用を実現する。

| Phase | 目的 | 主な成果物 |
|-------|------|-----------|
| 0 事前整理・方式確定 | 実現可否とPoC成功条件の明確化 | 要件定義/構成案/評価指標/「99.99%」「1秒」の定義合意 |
| 1 現場調査・撮影設計 | 安定撮影できる条件の確立 | 撮影環境設計書/カメラ・照明候補/撮影手順書/画質基準 |
| 2 PoCデータ構築 | 数百品番のデータセット | 学習/評価分割・類似グループ表・撮影条件メタ |
| 3 AIモデル開発 | 安定した特徴ベクトル抽出 | バックボーン選定・距離学習・評価レポート・閾値設計 |
| 4 検索・照合API | 高速照合と連携 | 索引・照合API・C#連携サンプル・API仕様 |
| 5 管理画面・C#連携 | 現場で使える形 | 管理UI・既存連携・照合履歴・FB機能 |
| 6 本番化・最適化 | 2万品番/1秒/高信頼 | DINOv2・Milvus・gRPC・ONNX/TensorRT・性能試験 |
| 7 現場導入・改善 | 実ライン運用 | 試験運用・誤判定分析・教育・継続改善計画 |

## 6.2 本リポジトリの現状（実装済み）
Phase 0〜5 を**動作する形で実装済み**：
- 差替可能バックボーン（classic/torchvision/**DINOv2 対応コード**）
- 距離学習（ArcFace 投影ヘッド、学習=torch/推論=numpy）
- ベクトル検索（Faiss + numpy フォールバック）
- 安全側判定エンジン（OK/NG/REVIEW/RETAKE/UNKNOWN）
- 画質ゲート、合成データ生成、安全指標ベンチマーク
- FastAPI + 管理UI + SQLite ログ + フィードバック
- C#連携サンプル、PostgreSQL本番スキーマ、テスト19件

## 6.3 本番化で差し替える点（Phase 6）
| 項目 | PoC（本実装） | 本番 |
|------|---------------|------|
| バックボーン | torchvision CNN | **DINOv2 ViT**（`PARTMATCH_BACKEND=dinov2`） |
| ベクトルDB | Faiss(IndexFlatIP) | Milvus / Qdrant / pgvector、ANN索引 |
| API | FastAPI REST | gRPC（低遅延）、REST併存 |
| 推論最適化 | PyTorch(CPU) | ONNX Runtime / TensorRT、GPUバッチ |
| DB | SQLite | PostgreSQL（`database/schema_postgres.sql`） |
| 画像保管 | ローカル/NAS | MinIO(S3互換)/NAS |
| 監視 | stats API | Prometheus / Grafana、精度ドリフト検知 |
| モデル管理 | npz | MLflow / DVC |

## 6.4 マイルストーン例（着手〜納期で約10か月）
1. 現場調査・撮影設計・成功基準合意
2. 数百品番PoC（撮影・学習・Faiss照合・精度/速度評価）
3. PoC評価・方式確定（カメラ/照明/バックボーン/検索DB）
4. MVP（照合API・C#連携・管理画面・履歴・FB）
5. 2万品番拡張・本番化（ベクトルDB・gRPC・推論最適化・閾値管理・再学習基盤）
6. 現場テスト（実ライン・誤判定分析・撮影調整・性能試験）
7. 本番導入（チューニング・マニュアル・教育・引き渡し・保守）

## 6.5 リスクと対策（要点）
| リスク | 対策 |
|--------|------|
| 99.99%達成難度 | 安全ゲート＋要確認フロー、類似グループ管理、複数角度、補助情報（バーコード等）併用、False Positive最小化評価 |
| 撮影環境による精度低下 | 現地で撮影設計、ドーム/同軸/偏光、位置決め治具、画質ゲート、手順書化 |
| 学習データ不足 | 距離学習＋ギャラリ照合、データ拡張、少数品番の重点撮影、追加・再学習フロー |
| C#連携負荷 | 明確なAPI仕様、タイムアウト/リトライ/異常時処理、フェイルセーフ、両側ログ |
| 1秒以内 | GPU/ONNX/TensorRT、Faiss高速化、通信軽量化、早期ベンチマーク |
