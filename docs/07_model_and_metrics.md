# 07. モデルと評価指標

## 7.1 特徴抽出 → 距離学習 → 検索
```
画像 → backbone(凍結) → ベース埋め込み → 投影ヘッド(学習) → 256次元(L2正規化)
                                                   → Faiss 内積=コサイン検索
```

### バックボーン（差替可）
| backend | 次元 | 依存 | 位置づけ |
|---------|------|------|----------|
| classic | 507 | numpy/PIL | フォールバック/オフライン。HOG+Hu+強度grid+半径profile |
| torchvision | 512–576 | torch | 既定。ImageNet事前学習CNN |
| dinov2 | 384–1024 | torch+hub | **本番推奨**。自己教師ありViT基盤モデル |

### 距離学習（Metric Learning）
- 投影ヘッド: `Linear→BN→ReLU→Linear→L2正規化`
- 損失: **ArcFace**（加法的角度マージン）。クラス内を密・クラス間を疎にし類似品の境界を鋭く。
- 学習はギャラリ埋め込み（凍結バックボーン出力）に対して行うため CPU で数秒。
- **学習は torch、推論は npz から純 numpy** で適用（`metric_learning/head.py`）。
- クラス分類器は学習時のみ。**新規品番はギャラリ追加で対応**（再学習不要）。

## 7.2 評価指標の定義
| 指標 | 定義 | 重要性 |
|------|------|--------|
| Top-1 / Top-3 | 最近傍/上位3に正解品番が含まれる率 | 検索性能 |
| **誤受理率 False Accept** | 取り違え（実物≠期待）を **OKで通す**率 | ★最重要・誤出荷に直結 |
| **OK精度** | OK判定のうち実際に正しい率 | 自動化の信頼性 |
| 取り違え検出率 | 取り違えを NG/REVIEW で捕捉する率 | 安全性 |
| 要確認率(REVIEW) | 人手確認に回る率 | 運用コスト |
| 類似グループ内精度 | 同一グループ内だけでの識別率 | 最難ケース |
| 推論時間 | 前処理+抽出+検索+判定の合計 | 1秒要件 |

> **単純な Accuracy では不十分**。出荷ミス防止が目的のため、間違った品番をOKとする
> False Positive（誤受理）を極小化する評価設計とする。

## 7.3 実測値（本実装・ホールドアウト評価）
構成: `torchvision/mobilenet_v3_small` + ArcFace投影(256d) + Faiss、50品番/12グループ、
ギャラリ6枚・評価3枚、CPU。

| 指標 | 値 |
|------|----|
| Top-1 / Top-3 | 0.833 / 0.940 |
| 独立部品 Top-1 | 1.000 |
| 類似グループ Top-1 | 0.830 |
| **誤受理率** | **0.000** |
| **OK精度** | **1.000** |
| **取り違え検出率** | **1.000** |
| correct-load 内訳 | OK 69% / REVIEW 21% / NG(安全側) 9% |
| 推論 avg / p95 | 12.8 / 15.1 ms |

再現: `python scripts/30_benchmark.py --parts 50 --groups 12 --epochs 80`

### バックボーン比較（同一条件: 40品番/12グループ/gallery5/query3/epochs60, CPU）
| backbone | dim | Top-1 | Top-3 | 独立Top-1 | 誤受理率 | OK精度 | 取り違え検出率 | avg ms/件 |
|----------|-----|-------|-------|-----------|----------|--------|----------------|-----------|
| classic | 256 | 0.450 | 0.758 | 1.000 | 0.0083 | 0.977 | 0.992 | **3.9** |
| torchvision(mobilenet_v3_small) | 256 | 0.700 | 0.925 | 1.000 | 0.0083 | 0.986 | 0.992 | 33.7 |
| **dinov2(vit_s14)** | 256 | **0.833** | **0.967** | 1.000 | **0.000** | **1.000** | **1.000** | 147.9 |

- **DINOv2 が本番経路として最良**。この難条件で唯一 **誤受理0・OK精度1.0・取り違え検出率1.0** を達成。
- DINOv2 は CPU で 148ms/件でも 1秒要件内。GPU では一桁ms。classic は最速だがフォールバック向き。
- 切替は `.env` の `PARTMATCH_BACKEND=dinov2` のみ（推論コード共通）。

### 解釈
- 生 Top-1 が 83% でも **誤出荷は0件**。曖昧な分を REVIEW/NG に回すため「OKは必ず正しい」。
- 「correct-load の NG 9%」は**正しい部品を安全側で止めた**もので、誤出荷ではなく手動確認。
  撮影条件改善・DINOv2・しきい値最適化で低減できる。

## 7.4 精度を上げる施策（本番）
1. **DINOv2 バックボーン**：表現力が高く、特に類似品の分離が向上。
2. **マルチビュー照合**：正面/斜め/エッジ強調/シルエットなど複数画像で頑健化。
3. **品番別・グループ別しきい値**：高リスク品番を厳格化。
4. **複数枚登録 + 品番集約**：撮影ばらつきを吸収（実装済み）。
5. **継続学習**：現場フィードバックで投影ヘッドを定期再学習。
6. **補助情報併用**：バーコード/工程情報をルールで併用し最終OKを二重化。

## 7.5 信頼度の扱い
- コサイン類似度 `[-1,1]` を表示用に `confidence=(cos+1)/2` へ写像。
- 判定はコサイン素の値としきい値で行う（`accept/review/margin/similar_margin`）。
- 本番では検証データで**しきい値を最適化**し、許容 REVIEW 率の下で誤受理率0を担保する点を選ぶ。
