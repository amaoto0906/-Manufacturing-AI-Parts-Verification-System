# 01. 要件定義

## 1.1 背景・目的
自動車部品メーカーの製造現場において、約2万種類の金属プレス部品を扱う。出荷時に品番を
取り違えると重大なクレーム・リコールにつながるため、**画像でAI照合し誤出荷を限りなくゼロに
する**ことが目的。汎用の外観検査（良否判定）ではなく、**多品種・類似品からの品番特定（照合）**
タスクである。

## 1.2 スコープ
- 撮影画像から品番を照合し、作業指示（期待品番）との一致を判定
- 似た部品の取り違えを「OKで通さない」ことを最優先（安全側設計）
- 既存C#システム（出荷管理・PLC・設備）との連携
- 品番追加・再学習・精度監視を含む運用基盤

## 1.3 機能要件
| ID | 要件 |
|----|------|
| F-01 | 撮影画像と期待品番を受け取り、OK/NG/REVIEW/RETAKE/UNKNOWN を返す |
| F-02 | Top-K 候補と類似度・信頼度・Top1-Top2マージンを返す |
| F-03 | 画質ゲートで不良画像を照合前に除外し再撮影を促す |
| F-04 | 品番マスタ・登録画像・類似品番グループを管理 |
| F-05 | 品番別・グループ別に判定しきい値を設定可能 |
| F-06 | 照合ログ（判定・画像・スコア・作業者・ライン・時刻）を保存 |
| F-07 | 誤判定フィードバックを蓄積し再学習データに回す |
| F-08 | 新規品番を**モデル全体の再学習なしに**登録画像追加だけで照合対象化 |
| F-09 | 既存C#システムから REST/gRPC で照合を依頼可能 |
| F-10 | 精度・要確認率・誤判定率のダッシュボード可視化 |

## 1.4 非機能要件
| ID | 要件 | 本実装の到達点 |
|----|------|----------------|
| N-01 | 推論1秒以内（撮影→結果） | CPU で avg 12.8ms / p95 15.1ms |
| N-02 | 2万品番規模のベクトル検索 | Faiss（→Milvus/Qdrant）で線形拡張可 |
| N-03 | **誤出荷（誤受理）率 ≈ 0** | ベンチで 0.000 |
| N-04 | 新規品番追加の容易さ | embedding 追加のみ・再学習不要 |
| N-05 | 撮影条件変化へのロバスト性 | 画質ゲート + 距離学習 + 複数枚登録 |
| N-06 | 可用性（AI停止時） | C#側フェイルセーフ（自動OK禁止→要確認） |

## 1.5 「99.99%」「1秒以内」の定義（要クライアント合意）
- **99.99%**：単純な Top-1 精度ではなく「**誤出荷率（間違った品番をOKで通す率）**」で定義する
  ことを推奨。AI単体の Top-1 ではなく、安全ゲート後の **OK精度 / 誤受理率** をKPIとする。
- **1秒以内**：本実装は「撮影画像受領→判定返却（前処理+特徴抽出+検索+判定）」を対象に計測。
  通信・カメラ露光時間は別途現場計測。

## 1.6 評価指標（詳細は docs/07）
Top-1 / Top-3 正解率、誤受理率（False Accept）、OK精度、取り違え検出率、要確認率、
類似品番グループ内精度、1件あたり推論時間。

## 1.7 前提・制約
- 本番では撮影条件を治具・照明で固定できることが精度の前提
- 一部品番は学習画像が少ない可能性 → 距離学習＋ギャラリ照合で対応
- 金属反射が強い → 画質ゲートと照明設計（ドーム/同軸/偏光）で対策
